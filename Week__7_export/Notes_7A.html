Root-Finding

polyroot(): input a vector of polynomial coefficients in increasing order, and get roots out. Output is given as complex numbers; real roots have imaginary coefficients of 0.

polyroot(c(1,-2,3,3,-1)) # find roots of 1 - 2*x + 3*x^2 + 3*x^3 - x^4
## [1]  0.2770792+0.3766347i  0.2770792-0.3766347i -1.2407239+0.0000000i
## [4]  3.6865656-0.0000000i
# This has two complex roots and two real roots.

uniroot(): a more general variant of polyroot() that searches for a root of a given function over an interval. However, it only â€œworksâ€ if there is one root within the interval given. If there are an even number of roots, uniroot() issues an error message. If there are an odd number of roots, only one is output.

f = function(x) {
  return(cos(x^2))
}
uniroot(f=f,interval=c(0,pi/2))$root # remember: pi is a reserved constant within R
## [1] 1.253314
x = seq(0,pi/2,by=pi/100)
plot(x,f(x),typ="l")
abline(h=0,col="chartreuse")
abline(v=uniroot(f=f,interval=c(0,pi))$root,col="chartreuse")

Approximation/Interpolation: Linear

For a given set of \((x,y)\) values, what is the value of \(y\) associated with \(x=x_o\)? Note that this is done via linear interpolation, which can lead to accurate enough results if the function is densely sampled.

x = seq(0,pi,by=pi/100)
y = cos(x^2)
# what is the value of this function at x = 2 and x = 3?
approx(x,y,xout=c(2,3))
## $x
## [1] 2 3
## 
## $y
## [1] -0.6523103 -0.9071864

Approximation/Interpolation: Splines

A â€œsplineâ€ is a piecewise polynomial function (often utilizing cubic polynomials). It is a typical â€œnonparametricâ€ estimator of an underlying function. There is much background material available via the web on splines; for our purposes here, it suffices to say it is a more flexible approximator than linear interpolation.

y = cos(x^2)
# what is the value of this function at x = 2 and x = 3?
spline(x,y,xout=c(2,3))
## $x
## [1] 2 3
## 
## $y
## [1] -0.653643 -0.911128

Integration

Letâ€™s say you wish to evaluate this integral: \[ \int_{-\pi/4}^{\pi/4} \left(1 + \frac{1}{\cos(x)}\right) dx \]

To do this, code it, then call the R function integrate():

integrand = function(x) {
  return(1+1/cos(x))
}
integrate(integrand,-pi/4,pi/4)
## 3.333544 with absolute error < 2.9e-13

If you just want the number, access the value list element.

integrate(integrand,-pi/4,pi/4)$value
## [1] 3.333544

Differentiation

Computing and evaluating a derivative is kind of a weird one, in the way that the process is structured. Here weâ€™ll just do a simple univariate example. See the help page for deriv for more (somewhat obtuse) detail.

f = expression(exp(x^2)*cos(x))   # put the function into a call to "expression"
(df = D(f,"x"))                   # evaluate the derivative symbolically...and save the output
## exp(x^2) * (2 * x) * cos(x) - exp(x^2) * sin(x)
x = seq(0,1,by=0.1)               # set the values of x over which to evaluate the derivative
eval(df)                          # evaluate the derivative...note, x is not included in call!
##  [1] 0.0000000 0.1001641 0.2012484 0.3038322 0.4077134 0.5112437 0.6102547
##  [8] 0.6962831 0.7536134 0.7543328 0.6500326
